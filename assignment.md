# TRIAL PROJECT: TASK MANAGEMENT

# SYSTEM

## Objective

The objective of this project is to build a simple Task Management System that allows users
to create, view, update, and delete tasks. The system will provide features like marking tasks
as complete or incomplete and filtering tasks based on their status. This project assesses
skills in backend development with .NET Core, frontend development with React, database
interaction, API integration, and deployment readiness.

## Requirements

## Backend (.NET Core)

The backend will be developed using .NET Core Web API. The focus will be on building
RESTful APIs, managing database interactions, and implementing basic validation and
documentation.

- Create a RESTful API with endpoints for CRUD operations on tasks.
- Use a database (SQL Server or SQLite) to store task data.
- Implement data validation for incoming requests.
- Provide Swagger (OpenAPI) documentation for the API.

## Frontend (React)

The frontend will be developed using React to provide an intuitive and responsive user
interface. This includes pages for task listing, adding/editing tasks, and filtering tasks based
on status.

## General Requirements

This section outlines additional features and considerations for the project, such as
implementing optional authentication, deployment readiness, and ensuring high code
quality.

- Implement basic authentication ( **optional** ).
- Containerisation of services (database if using sql server, front-end, back-end)
    ( **optional** ).
- Documentation on how to run and access the services
- Follow clean coding practices and include meaningful comments.

## Evaluation Criteria

The evaluation criteria focus on assessing technical skills, problem solving ability, code
quality, and deployment readiness. Bonus points will be given for additional features like
unit tests or advanced functionality.

- Technical Skills: Backend API design, database interaction, and frontend integration.
- Code Quality: Maintainable, well documented, and clean code.
- Problem Solving: Effective handling of edge cases and logical implementation.
- Deployment Readiness: Successfully deployed and accessible project.
- Bonus Points: Unit tests, task prioritization, or ‘drag and drop’ task sorting,
    authentication, containerisation
